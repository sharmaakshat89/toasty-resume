RESUME OPTIMIZATION — DETAILED (LLM-READY FORMAT)

CORE MODEL
Resume = High-signal, keyword-optimized, fast-scan document designed for human screening (NOT ATS parsing).

PRIMARY CONSTRAINT
HR scan time ≈ 3–5 seconds
→ Resume must communicate value instantly at a glance.

ATS REALITY
- ATS = storage + tracking system
- Does NOT reliably parse or rank resumes (in most cases)
- Do NOT optimize for ATS tricks
- Optimize for human keyword scanning instead

---

OBJECTIVE FUNCTION
Maximize:
- Signal density (useful info per line)
- Keyword relevance (match job description)
- Scan clarity (easy to skim quickly)

Minimize:
- Cognitive load
- Visual clutter
- Irrelevant content

---

STRUCTURE (STRICT ORDER)

1. HEADER
   - Full Name (large, bold)
   - Email
   - Phone number
   - Location
   - GitHub / Portfolio / LinkedIn

2. TECHNICAL SKILLS
   - List only relevant, job-aligned skills
   - Group logically (e.g., Languages, Frameworks, Tools)

3. WORK EXPERIENCE (MOST IMPORTANT)
   - Reverse chronological order (latest first)
   - Focus on impact, not responsibilities
   - Prioritize relevance to target role

4. PROJECTS / ACHIEVEMENTS / CERTIFICATIONS (OPTIONAL)
   - Include ONLY if:
     → Strong
     → Relevant
     → Defensible in interview
   - Otherwise omit and expand work experience

5. EDUCATION
   - Reverse chronological
   - Minimal details

---

LAYOUT RULES

- Single column (MANDATORY)
- Top-to-bottom reading flow
- No sidebars, no split layouts
- No complex formatting

---

TYPOGRAPHY

Allowed fonts:
- Times New Roman
- Arial
- Any serif-family equivalent

Font size:
- Body: 10–11 pt
- Headers: larger

---

STYLING RULES

Allowed:
- Bold → section headers, key emphasis
- Italics → secondary info
- Blue + underline → hyperlinks

Forbidden:
- Colors (except links)
- Icons, graphics, visual elements
- Fancy templates
- Decorative formatting

Design principle:
KISS = Keep It Simple and Structured

---

CONTENT RULES

GENERAL
- Every line must justify its existence
- Remove fluff, filler, vague claims

SKILLS
- Must align with job description keywords
- Avoid generic terms (e.g., “hardworking”)

WORK EXPERIENCE
- Highest weight section
- Emphasize:
  → Impact
  → Outcomes
  → Relevance

PROJECTS
- Include only if:
  → You can explain deeply in interview
- If weak → REMOVE

EDUCATION
- Keep concise
- No unnecessary detail

---

HR SCANNING MODEL

Scan path:
Header → Skills → Experience → Decision

Decision criteria:
- Keyword match
- Clarity
- Relevance

Time budget:
~3 seconds

Implication:
→ Important information must be visible immediately

---

OPTIMIZATION STRATEGY

- Front-load strongest signals (top sections matter most)
- Use keywords from job description
- Make resume skimmable (not dense blocks)
- Ensure quick pattern recognition

---

MAINTENANCE SYSTEM (BRAGDOC)

Process:
1. Maintain ongoing log of achievements (Bragdoc)
2. Update frequently (daily or weekly)
3. Every ~3 months:
   → Extract best entries
   → Update resume

Benefits:
- No forgotten achievements
- Always up-to-date resume

---

QUALITY CONTROL

Grammar:
- Zero visible errors
- Use tools:
  → Grammarly
  → Hemingway
  → LLM review

Peer review:
- External feedback improves quality

---

JOB SEARCH META

- Job search = numbers game
- Requires:
  → High application volume
  → Consistency
  → Patience

Networking:
- Do not directly ask for jobs
- Build context before requests

---

HARD RULES SUMMARY

DO:
- Optimize for human scanning
- Use standard structure
- Use serif fonts
- Keep layout linear
- Prioritize work experience
- Align with keywords

DO NOT:
- Use columns
- Overdesign
- Add fluff
- Rely on ATS parsing
- Include weak/unjustified content
